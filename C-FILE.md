### **C语言的文件操作**

### 📂**打开文件**：`FILE *fopen( const char *filename, const char *mode );`
在这里，**filename** 是字符串，用来命名文件，访问模式 **mode** 的值可以是下列值中的一个：
- **r**，打开一个已有*文本文件*，并且只允许读取，若文件不存在则会报错
- **r+**，打开一个*文本文件*，允许读写文件。<mark style="background: #FF5582A6;">读取不会清空原文件内容，但写入会从头开始覆盖</mark>
- **w**，打开一个*文本文件*，允许写入。若文件不存在，则会创建一个新文件。<mark style="background: #FF5582A6;">若文件存在，则会清空原文件内容并打开（或者理解为重新创建一个空文件覆盖原文件，总之原文件内容将不复存在）</mark>
- **w+**，打开一个*文本文件*，允许读写文件。若文件不存在，则会创建一个新文件。<mark style="background: #FF5582A6;">若文件存在，则会清空原文件内容并打开（或者理解为重新创建一个空文件覆盖原文件，总之原文件内容将不复存在）</mark>
- **a**，打开一个*文本文件*，以追加模式写入文件。若文件不存在，则会创建一个新文件。
- **a+**，打开一个*文本文件*，允许读写文件。若文件不存在，则会创建一个新文件。<mark style="background: #FF5582A6;">读取会从文件头开始，写入则为追加模式</mark>  

👉**若为二进制文件，只需在末尾加上`b`，如：`rb`,`rb+`,`r+b`（后两个写法等价）**

### 📴**关闭文件**：` int fclose( FILE *fp );`
⚠️建议这样做，原因如下：

 推荐 `fclose(pf); pf = NULL;`

#### 主要原因

**防止悬空指针（Dangling Pointer）**

`fclose(pf)` 关闭文件并释放相关资源后，`pf` 仍然保存着原来的地址值，成为悬空指针。若之后误操作：

```c
fclose(pf);
// 忘记已关闭，再次使用
fwrite(buf, 1, len, pf);  // 未定义行为，可能崩溃
fclose(pf);               // 二次关闭，未定义行为
```

赋值 NULL 后，误操作会触发可检测的错误：

```c
fclose(pf);
pf = NULL;

if (pf != NULL) {         // 或直接 if (pf)
    fwrite(buf, 1, len, pf);
}
```


#### 什么时候可以不赋 NULL

| 场景 | 是否需要赋 NULL |
|---|---|
| 关闭后指针立即离开作用域 | 不必要 |
| 指针在函数末尾关闭 | 意义不大 |
| 指针可能被多处复用或复用判断 | **强烈建议** |
| 多次 fclose 风险存在 | **强烈建议** |

```c
// 此场景赋 NULL 意义不大——指针马上消亡
void foo() {
    FILE *pf = fopen("a.txt", "r");
    // ... 使用 ...
    fclose(pf);
    // 函数结束，pf 自动销毁
}
```


#### 最佳实践封装

```c
void safe_fclose(FILE **pp) {
    if (pp && *pp) {
        fclose(*pp);
        *pp = NULL;
    }
}

// 使用
safe_fclose(&pf);
```

这样同时解决了：
- 空指针传入保护
- 关闭后自动置 NULL
- 防止二次关闭


**结论：** 这是一个良好的防御性编程习惯，在指针生命周期较长或存在复用可能时尤其值得坚持。
